# Task Plan

## Goal

梳理 DocGen 当前全部项目文档，识别重复、过时和缺失内容；基于新的产品方向重建一套完整的项目文档结构，并落地核心文档更新。

## Phases

| Phase | Status | Description |
|-------|--------|-------------|
| 1 | completed | 盘点现有文档并建立分类视图 |
| 2 | completed | 识别文档问题、重叠和与新策略不一致之处 |
| 3 | completed | 设计新的完整项目文档结构 |
| 4 | completed | 更新核心文档内容并新增缺失文档 |
| 5 | completed | 校对一致性并给出后续维护建议 |
| 6 | in_progress | 拆解下一阶段功能并形成可执行开发任务文档 |

## Decisions

- 优先保留已有文件路径，除非存在明显职责冲突。
- 在不影响用户现有代码改动的前提下，尽量通过新增和改写文档完成本轮工作。
- 新的文档体系将围绕“技术教程 / 学习指南 / 技术手册生成工具”这一定位组织。

## Risks

- 当前仓库存在未提交改动，修改现有文档时需要避免覆盖与本任务无关的工作。
- 部分状态类文档可能互相矛盾，需要统一口径。
